Javascript lives within an HTML web page

Of course there is the NodeJS exception but for the purpose of these tutorials we consider the client side usage of JS.

So then Javascript works with the HTML document that contains or loads the script.

This sentence implies both benefits and restrictions (most of them are tied to secutiry reasons) than we need to learn and understant in order to get the most efficient development experience.

Embedded Javascript

The right place where to put your first Javascript instruction is a <script> tag: 

<body>
    <script>
        alert('I am Javascript');
    </script>
</body>

You can place such tag in both <head> and <body> but keep in mind that any Javascript code you place into the document's <head> will take some execution time which delays the page rendering.

Linked Javascript

When your code grows big you may find a good practice to move it to dedicated .js files which you link within the document: 

<body>
    <script src="./path/to/my/script.js"></script>
</body>

This is the classic approach to link libraries such jQuery.
If you place such instructions into the document's <head> you'll delay the page rendering of both download time + execution time.

Always remember that the best place for Javascript is the document's body close to the </body> tag!
